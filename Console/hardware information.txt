# Hardware notes for myself and others who join the project.
These notes are mostly for reverse engineering purposes

Turns out I was mistaken:
There's just one ADC line, and all 74HC4051 share the same analog bus, while their /EN lines are used to select which chip to read.
So, effectively, they form an 8x3 matrix
where the 8 columns are select using pins S0, S1 and S2
and the row is selected using the respective chip's /EN line

There's
PortA: 8-bit IO
and PortB: 8-bit O only

Here's where I need to question if it makes any sense to plug our RP2040 directly to each chip and matrix
or if we should use the original circuitry including the HC245 and HC573 array IO "controller"

There are 6 extra chips
* one hc245 (U3) and five hc573 (U5, U11, U4, U16 and U2)

* U3  Is the button reader
* U5  Drives the LED columns (assuming a row is a contiguous array of LEDs)
* U11 Is the MUX select pins (D0, D1, D2 = S0, S1, S2; D3, D4, D5 = IC0_/EN, IC1_/EN, IC2_/EN) U11 selects which fader to read
* U4  Drives the LED rows (drives transistors)
* U16 Is the button driver (Has output diodes, so a given scanning row/column is active LOW. However, it does not have anti-ghosting diodes... so ghosting is possible... though it may not require handling, since the user will almost never use more that one or two buttons at once, and ghosting is critical only at 3+ buttons begin held)
* U2  Seems to be related to the original CPU's RAM chip (It used discrete CPU, RAM and ROM ICs)

This means it is likely U11 drives... uh... idk
